MySQL安装指南

MySQL 是最流行的关系型数据库管理系统，在 WEB 应用方面 MySQL 是最好的 RDBMS(Relational Database Management System：关系数据库管理系统)应用软件之一。

本篇记录一下在Ubuntu 20.04系统上配置MySQL的过程。

# 1.查看是否安装过

rpm -qa | grep mysql

# 彻底卸载
sudo apt purge mysql-*
sudo rm -rf /etc/mysql/ /var/lib/mysql
sudo apt autoremove
sudo apt autoclean

# 2.安装MySQL

# 更新软件源并安装
sudo apt-get update
sudo apt-get install mysql-server

# 3.配置MySQL

# 启动服务
systemctl start mysqld

# 查看服务状态
systemctl status mysql
netstat -tap | grep mysql

# 运行mysql初始化安全脚本
sudo mysql_secure_installation

设置VALIDATE PASSWORD PLUGIN插件来测试MySQL用户密码强度并提高安全性的密码，输入y

VALIDATE PASSWORD COMPONENT can be used to test passwords
and improve security. It checks the strength of password
and allows the users to set only those passwords which are
secure enough. Would you like to setup VALIDATE PASSWORD component?

Press y|Y for Yes, any other key for No: y

密码验证策略分为三个级别：低，中和强

There are three levels of password validation policy:

LOW    Length >= 8
MEDIUM Length >= 8, numeric, mixed case, and special characters
STRONG Length >= 8, numeric, mixed case, special characters and dictionary

Please enter 0 = LOW, 1 = MEDIUM and 2 = STRONG: 0

注意

MySQL设置密码时显示... Failed! Error: SET PASSWORD has no significance for user 'root'@'localhost' as the authentication method used doesn't store authentication datatead if you want to change authentication parameters.

解决方案

  1. sudo mysql进入MySQL
  2. ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password by '新密码';
  3. exit退出MySQL，输入： sudo mysql_secure_installation
  4. 输入刚刚的新密码，出现的所有问题都n即可。

# 4.配置新角色权限

create user 'administrator'@'%' identified by 'administrator';
grant all privileges on *.* to 'administrator'@'%';

# 5.修改root账户密码

MySQL使用的是validate_password插件（plugin）检测、验证账号密码强度，保障账号的安全性

set global validate_password.policy=0; # 密码强度检验等级 0: 只检验长度
set global validate_password.length=4; # 密码最小长度
set global validate_password.mixed_case_count=0; # 密码具有大写和小写字符的最小数量
set global validate_password.check_user_name=false; # 密码可以设置为当前用户名
flush privileges;

ALTER USER 'root'@'localhost' IDENTIFIED BY 'root';
